# КАВАЧАМ — публичный слой

Публичная одностраничная презентация проекта **КАВАЧАМ** (kavacham.ai) — ИИ-защита чистоты учения Шрилы Прабхупады: отвечает только дословными цитатами с проверяемыми ссылками.

- **Живая ссылка:** https://dgdbvgm-tech.github.io/kavacham/
- **Формат:** самодостаточный статический HTML (без сборки, без внешних зависимостей в рантайме).
- **Хостинг:** GitHub Pages.

## Страницы

| Путь | Что это |
|---|---|
| `index.html` | Питч-презентация: что за проект, как работает, чем помочь |
| `constitution.html` | Конституция агента — законы, по которым он отвечает (и молчит) |

| `styles.css` | Общая дизайн-система всех поверхностей: палитра, типографика, тёмная/светлая тема |

## Приложение-читальня (`/app/`)

`/app/` — **«Читальня КАВАЧАМ»**, Telegram Mini App поверх бота Лаборатории: разборы и мега-разборы открываются **одной страницей** (без ограничения в 64 КБ, из-за которого длинные разборы приходилось резать на части), с оглавлением, кликабельными праманами и тир-разметкой источников.

- Статический фронт: официальный SDK Telegram лежит рядом (`app/vendor/`), **внешние CDN в рантайме не используются**.
- Тема примиряется с темой клиента Telegram (тёмная/светлая), поверх фирменной палитры проекта.
- Контент читальни — данные в `app/data/`: лента (`feed.json`) и тело каждого разбора (`data/reading/<slug>.json`).

**Ссылка-приложение** (кнопка «Читальня» в боте) активируется после подключения в BotFather — до этого момента живой ссылки нет, и мы её не публикуем.

## Статические страницы разборов (`/reading/`)

`/reading/<slug>.html` — тот же разбор **самодостаточной веб-страницей**, открывается в любом браузере, без Telegram.

Это сознательный второй путь к тому же тексту: в ряде регионов мини-приложения Telegram не открываются, и читатель не должен терять доступ к разбору. Обе поверхности — читальня и статическая страница — генерируются **из одного исходника**, поэтому текст и ссылки на них совпадают.

## Статус

Проект **в стадии песочницы**: лаборатория для тех, кто присылает вызовы и проверяет ответы движка, а не готовый продукт. Разборы публикуются с инвариантом «ноль энтропии»: дословная цитата + проверяемая ссылка на источник; нет ссылки — значит это честно помеченный пробел, а не догадка.
