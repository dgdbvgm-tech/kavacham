# КАВАЧАМ — публичный слой

Публичная одностраничная презентация проекта **КАВАЧАМ** (kavacham.ai) — ИИ-защита чистоты учения Шрилы Прабхупады: отвечает только дословными цитатами с проверяемыми ссылками.

- **Живая ссылка:** https://dgdbvgm-tech.github.io/kavacham/
- **Формат:** самодостаточный статический HTML (без сборки, без внешних зависимостей в рантайме).
- **Хостинг:** GitHub Pages.

## Страницы

| Путь | Что это |
|---|---|
| `index.html` | Питч-презентация: что за проект, как работает, чем помочь |

| `styles.css` | Общая дизайн-система всех поверхностей: палитра, типографика, тёмная/светлая тема |

## Приложение «KAVACHAM Lab» (`/app/`)

`/app/` — **«KAVACHAM Lab»** — Telegram Mini App поверх бота Лаборатории. Материалы открываются **одной страницей** (без ограничения в 64 КБ, из-за которого длинные разборы приходилось резать на части), с оглавлением, кликабельными праманами и тир-разметкой источников.

**Экраны:**

| Экран | Что делает |
|---|---|
| **Полигон (Протоколы)** | Лента всех опубликованных материалов по рубрикам: **Разборы · Мега-разборы · Аналитика · Дневник разработки · Карты**. Материал открывается страницей с оглавлением и ссылкой на каждую цитату |
| **Приёмная** | Прислать вызов на разбор: текст вопроса → выбор источников (основа — Шрила Прабхупада, сверху по желанию ачарьи парампары, документы GBC, учители в линии) → имя или анонимно → в очередь |
| **Кабинет** | «Мои заявки»: номер, статус, честное место в очереди (разборы выходят строго по порядку подачи), ссылка на готовый разбор |
| **О проекте** | Что это, принципы (ноль энтропии, право молчать), ссылки на витрину проекта и канал Лаборатории |

- **Анонимность по умолчанию:** в публичном разборе заявитель — «участник Лаборатории». Имя указывается только по явному согласию, данному при отправке заявки.
- Статический фронт: официальный SDK Telegram лежит рядом (`app/vendor/`), **внешние CDN в рантайме не используются**. Данные — лента (`app/data/feed.json`) и тело каждого материала (`app/data/reading/<slug>.json`); заявки и статусы приходят из API бота.
- Тема примиряется с темой клиента Telegram (тёмная/светлая), поверх фирменной палитры проекта.

**Ссылка-приложение** (кнопка «Протоколы» в боте) активируется после подключения в BotFather — до этого момента живой ссылки нет, и мы её не публикуем.

## Статические страницы материалов (`/reading/`)

- `/reading/` — **оглавление протоколов**: все материалы по рубрикам, обычной веб-страницей.
- `/reading/<slug>.html` — сам материал **самодостаточной страницей**, открывается в любом браузере, без Telegram.

Это сознательный второй путь к тому же тексту: в ряде регионов мини-приложения Telegram не открываются, и читатель не должен терять доступ ни к одному материалу. Правило простое — **у любой возможности приложения есть равноценный путь через обычного бота и обычную веб-страницу**. Обе поверхности — протоколы и статические страницы — генерируются **из одного исходника**, поэтому текст и ссылки на них совпадают.

## Статус

Проект **в стадии песочницы**: лаборатория для тех, кто присылает вызовы и проверяет ответы движка, а не готовый продукт. Разборы публикуются с инвариантом «ноль энтропии»: дословная цитата + проверяемая ссылка на источник; нет ссылки — значит это честно помеченный пробел, а не догадка.
