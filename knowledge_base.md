## Что такое KAVACHAM
KAVACHAM (санскр. «броня», от Нараяна-кавачам, ШБ 6.8; акроним Knowledge-Anchored Vaishnava Apologetics & Citation Hub with Authoritative Mantras) — ИИ-агент защиты учения Шрилы Прабхупады в интернете. Он отвечает только дословными цитатами из проверенного корпуса первоисточников и всегда приводит кликабельную ссылку на источник. Это цифровой библиотекарь-апологет, а не «умная нейросеть» и не наставник.

## Что такое GJK
GJK (Gaudiya Jnana Kosh) — фундамент проекта: локальная библиотека материалов ИСККОН в открытом формате знаний (OKF: markdown + структурированные метаданные). На июль 2026 в ней более 38 000 документов, 0 ошибок валидации. KAVACHAM — прикладной модуль-апологет поверх этой библиотеки.

## Какую проблему решает продукт
Люди уже спрашивают обычный ИИ о сознании Кришны, а он выдумывает: приписывает Шриле Прабхупаде слова, которых он не говорил, смешивает искажения (ритвик, майявада), берёт переводы с ошибками. Проверить нельзя. KAVACHAM устраняет это: только выверенные источники, дословные цитаты, проверяемые ссылки, и честное «нет основания» вместо выдумки.

## Чем KAVACHAM лучше обычной нейросети (ИИ)
Обычная нейросеть (ChatGPT, обычный ИИ) звучит уверенно, но выдумывает: приписывает Шриле Прабхупаде слова, которых он не говорил, смешивает искажения (ритвик-вада, майявада), сочиняет несуществующие «цитаты» и ссылки, берёт тексты из агрегаторов с ошибками перевода — и никогда не скажет «я не знаю». Проверить нельзя. KAVACHAM устроен иначе: (1) отвечает только дословными цитатами из выверенных первоисточников и всегда даёт кликабельную ссылку — каждое слово проверяемо; (2) берёт только источники в парампаре, не смешивая учение с искажениями; (3) три рубежа проверки чистоты — от источника до ответа; (4) имеет право молчать: честное «нет основания» вместо фабрикации. Коротко: обычный ИИ оптимизирован звучать убедительно, KAVACHAM — быть точным и проверяемым. Он не «умная нейросеть», а цифровой библиотекарь-апологет.

## Как это работает — три принципа
1. Только дословные цитаты с точной ссылкой на первоисточник. Нет опоры в библиотеке — нет утверждения. 2. Право молчать: если ответа в источниках нет, агент честно говорит «в моих источниках нет основания» вместо фабрикации. 3. Не гуру и не вани: KAVACHAM подаёт точную книгу, но не заменяет живого общения и наставничества.

## Три рубежа чистоты
Чистота гарантируется тремя фильтрами: (1) Источник — белые/серые/чёрные списки, доверяем только выверенным хранилищам вани (Vanisource/BLS, издания BBT), отвергаем агрегаторы с ошибками; (2) Документ — у каждого текста статус проверки (ожидает / сверен владельцем / сверен с официальным изданием); (3) Ответ — по умолчанию только из доверенного контура, спорное уходит на ревью человека, публикация — с одобрения человека.

## SRI — Карта разрешённых парадоксов
Для известных сложных узлов сиддханты (например, вопрос о происхождении обусловленности души) Конституция v1.3 (§3c) запрещает движку свободный синтез: такие узлы отвечаются только по вручную выверенным эталонам SRI (Siddhanta Resolution Index) — подборкам, где собраны праманы всех линий высказываний, гармонизирующие документы самого Шрилы Прабхупады и официальная позиция ИСККОН, — либо движок честно молчит и направляет к живому наставнику. При этом система не выносит вердикт по спорному узлу (§4.8): она прозрачно показывает уровни источников, а решение остаётся за живой парампарой. Первый узел SRI создан по материалам разбора №1.

Векторизация — построение смыслового индекса для мгновенного точного поиска. ## Векторизация и «фрагменты» — что это и зачем
Каждый документ делится на небольшие смысловые фрагменты (по абзацам/стихам) и превращается в числовые «отпечатки смысла» — векторы. Поэтому фрагментов в разы больше, чем документов: один документ даёт много точек поиска. Зачем: поиск идёт по смыслу, а не по совпадению слов — система находит нужные места первоисточников, даже если формулировка вопроса другая, и приводит их дословно со ссылкой. Мелкий фрагмент = точное попадание в нужный стих или абзац, а не «вся книга целиком». Это стандарт индустрии (retrieval) для честного поиска по большим корпусам; для нас — фундамент точности и проверяемости, основа «трёх рубежей чистоты».

## Техническая архитектура
Три независимых слоя: (1) Библиотека — markdown, переносима, не привязана ни к одной нейросети (источник истины, ~20 ГБ «холодного» сырья); (2) Векторный индекс — компактный (~1–2 ГБ), обслуживает поиск, в проде выносится в управляемую вектор-БД (например Qdrant Cloud); (3) RAG-сервис — небольшой API (FastAPI на Google Cloud Run), собирает ответ с цитатами; фронт — статика (Netlify/GitHub Pages) + Telegram-бот. Стек переиспользует то, что у автора проекта уже есть: Netlify, Cloud Run, Python/aiogram, GitHub.

## Подключение мощной нейросети и голоса
Для глубоких аналитических ответов (в том числе разбор по логике ньяи) KAVACHAM подключает мощную LLM через API — это отдельный постоянный бюджет (оплата за токены). Голосовой ввод запросов уже работает на лендинге через браузерный Web Speech API (без ключей), а для более точного распознавания русской речи планируется/подключается Deepgram (речь → текст) через безопасный бэкенд-прокси. Ключи хранятся строго на сервере, не в браузере. Модель финансирования — см. секцию «Оплата и бюджет сервиса».

## Голосовой ввод и языки
Вопрос можно задать текстом или голосом. Голос на лендинге сейчас работает через браузерный Web Speech API (русский, ru-RU) — с живой расшифровкой в поле ввода и видимой индикацией записи. Для продукта планируется Deepgram (точнее распознаёт русскую речь, ставит пунктуацию). Понимание вопроса — многоязычное: русский вопрос находит английский стих (эмбеддинги multilingual-e5). Ключи распознавания — только на сервере.

## Оплата и бюджет сервиса
Библиотека и поиск построены разово и почти бесплатны (эмбеддинги считаются локально, $0; хостинг статики бесплатный). Постоянные расходы возникают у «глубокого» слоя — мощная LLM через API (центы за ответ, на объёме десятки-сотни $/мес). ## Почему KAVACHAM не спешит — точность превыше скорости («ноль энтропии»)
Скорость — не главная метрика KAVACHAM; главное — отсутствие ошибки. Быстрый неверный ответ хуже медленного верного: он крадёт самое дорогое — мгновение человеческой жизни, время, отпущенное на постижение Абсолютной Истины (каждый миг бесценен — ШБ 7.6.1, durlabham manusam janma). Поэтому агент готов думать минуты над сложным вопросом, но не выдаёт догадку. «Ноль ошибок, ноль галлюцинаций» держится на трёх опорах: (1) Заземление — каждое утверждение с проверяемой цитатой первоисточника (это архитектура, а не свойство модели, — настоящая защита от галлюцинации); (2) Глубина — столько проходов по источникам и вдумчивости, сколько нужно для верного разбора (на технологическом горизонте — парадигма RLM с рекурсивными подагентами); важно: «больше думать» само по себе не обнуляет уверенную ошибку, поэтому глубина лишь усиливает заземление, а не заменяет его; (3) Право молчать — честное «нет основания» вместо правдоподобной выдумки. Это и есть смысл подписи проекта ZERO ENTROPY: энтропия — мера шума и неопределённости, ноль энтропии — чистый сигнал без искажения; истину нельзя получить мгновенно и даром, за неё платят вдумчивой работой. ## Глоссарий терминов
Короткие пояснения терминов, которые встречаются в проекте и в ответах виджета.
- **Парампара** — цепь ученической преемственности, по которой знание передаётся от учителя к ученику без искажений (Б.-г. 4.2). KAVACHAM берёт источники только внутри парампары.
- **Сиддханта** — выверенный философский вывод, заключение писаний. Противоположность — апасиддханта.
- **Апасиддханта** — учение, расходящееся с выводом шастр и парампары. KAVACHAM создан, чтобы распознавать и разбирать апасиддханту (ритвик-вада, майявада, атеизм) методом Прабхупады.
- **Ритвик-вада** — отклонение, по которому после ухода Шрилы Прабхупады посвящение якобы даётся лишь формально «от его имени», без живого учителя; ИСККОН и GBC отвергают его как апасиддханту.
- **Майявада** — имперсоналистская философия (Абсолют в конечном счёте безличен), которую вайшнавская веданта опровергает.
- **Ньяя** — санскритская логика и искусство корректного рассуждения, методология доказательного разбора (одна из шести даршан).
- **Прамана** — источник достоверного познания. Высшая прамана здесь — шабда (свидетельство шастр), выше чувственного восприятия (пратьякша) и умозаключения (анумана).
- **Пурвапакша** — позиция оппонента, которую честно и в самой сильной форме излагают перед разбором (близко к принципу «steelman»).
- **Апологетика** — обоснованная защита учения средствами логики и первоисточников; профиль KAVACHAM.
- **Чатур-доша** — «четыре изъяна» обусловленного познания: бхрама (ошибка), прамада (невнимательность), випралипса (склонность обманывать), каранапатава (несовершенство чувств). В проекте — прообраз «галлюцинации» ИИ, который снимается опорой на шабда-праману.
- **Юкта-вайрагья** — «уместное отречение»: занимать любые средства, включая технологии, в служении, не привязываясь к ним (принцип Рупы Госвами).
- **Брихат-мриданга** — «большая мриданга» Бхактисиддханты Сарасвати: печатный станок и, шире, самый мощный канал эпохи как инструмент проповеди.
- **RAG** (Retrieval-Augmented Generation) — «поиск + генерация»: сначала находят точные фрагменты в корпусе, затем LLM отвечает строго по ним со ссылками. Так устроено ядро KAVACHAM.
- **RLM** (рекурсивные языковые модели) — следующий горизонт: агент с рекурсивными подагентами для многошагового разбора, где поиск и Конституция становятся его инструментами.
- **Эмбеддинги и векторизация** — перевод текста в числовые векторы смысла, чтобы искать по смыслу, а не по словам (модель multilingual-e5: русский вопрос находит английский стих).
- **OKF** (Open Knowledge Format) — открытый формат знания: markdown плюс структурированные метаданные; на нём построена библиотека GJK.
- **GJK** (Gaudiya Jnana Kosh) — «сокровищница знания гаудия»: локальная библиотека первоисточников, фундамент проекта.
- **Ноль энтропии** — кредо проекта: точность важнее скорости; честное молчание лучше правдоподобной ошибки.
